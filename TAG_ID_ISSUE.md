# 标签 ID 问题说明

## ❌ 错误：400 Bad Request - 标签是 ID 而不是文本

### 错误示例

```
GET /api/v1/pc-wallpapers?page=1&pageSize=16&tag=a754117de8bf 400 (Bad Request)
[推荐-调试] ❌ 请求失败: Error: Request failed with status code 400
```

### 问题原因

后端返回的 `tags` 字段是 **ID 数组**，而不是**可读的文本标签**。

**错误的数据格式** ❌:
```json
{
  "id": "1",
  "tags": ["a754117de8bf", "b863259cf1a2"],  // ← ID，不是文本
  "classificationTag": "c971348ad5e3"  // ← ID，不是文本
}
```

**正确的数据格式** ✅:
```json
{
  "id": "1",
  "tags": ["风景", "自然", "山水"],  // ← 文本标签
  "classificationTag": "自然|风景"  // ← 文本标签
}
```

### 1. 查看后端返回的原始数据

```bash

```

### 2. 检查 tags 字段的值

```json
{
  "data": {
    "items": [
      {
        "id": "1",
        "imageUrl": "...",
        "tags": ["???"]  // ← 看这里
      }
    ]
  }
}
```

**如果 tags 是这样的**:
```json
"tags": ["a754117de8bf", "b863259cf1a2"]
```
→ **问题**: 这些是 ID，不是标签文本

**应该改成这样的**:
```json
"tags": ["风景", "自然", "山水"]
```
→ **正确**: 这些是可读的文本标签

---

## 🛠️ 解决方法

### 方法 1: 后端返回标签文本（推荐）

修改后端代码，返回标签的**文本内容**而不是 ID：

```javascript
// ❌ 错误：返回标签 ID
const tags = wallpaper.tags.map(tag => tag.id)

// ✅ 正确：返回标签文本
const tags = wallpaper.tags.map(tag => tag.name)
```

**或者使用 JOIN 查询**:
```sql
-- 假设有 wallpaper_tags 关联表和 tags 表
SELECT w.*, GROUP_CONCAT(t.name) as tags
FROM wallpapers w
LEFT JOIN wallpaper_tags wt ON w.id = wt.wallpaper_id
LEFT JOIN tags t ON wt.tag_id = t.id
GROUP BY w.id
```

---

### 方法 2: 前端已添加过滤（临时方案）

前端现在会自动过滤掉疑似 ID 的标签：

```typescript
// 自动过滤以下情况：
// 1. 超过 20 个字符
// 2. 纯数字
// 3. 长字母数字组合（如 a754117de8bf）
// 4. 包含特殊字符
```

**但这只是临时方案，根本解决方法是修改后端返回正确的数据。**

## 📊 前端过滤规则

以下标签会被自动过滤：

| 标签示例 | 是否过滤 | 原因 |
|---------|---------|------|
| `"风景"` | ❌ 不过滤 | 有效的文本标签 |
| `"自然风景"` | ❌ 不过滤 | 有效的文本标签 |
| `"a754117de8bf"` | ✅ 过滤 | 疑似 ID（长字母数字） |
| `"12345"` | ✅ 过滤 | 纯数字 |
| `"verylongtextthatexceeds20characters"` | ✅ 过滤 | 超过 20 字符 |
| `"tag<script>"` | ✅ 过滤 | 包含特殊字符 |
