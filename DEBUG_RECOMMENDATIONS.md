# 推荐功能调试指南

## 🔧 已实施的优化

### 1. 降级策略：支持 classificationTag

如果壁纸没有 `tags` 字段，会自动尝试从 `classificationTag` 提取标签：

```javascript
// 示例：classificationTag: "自然|风景"
// 会被拆分为：["自然", "风景"]
```

支持的分隔符：`|` `/` `,` `，` `、`

### 2. 限制关键词数量

为避免同时发送过多请求导致页面卡顿：
- **最多保留前 3 个关键词**（按长度排序）
- **串行请求**，不并发（等待上一个请求完成再发下一个）
- **获得 16 条推荐后自动停止**

### 3. 简化关键词提取

- 只拆分 3-4 字的标签（减少关键词数量）
- 移除 3 字拆分（只保留 2 字拆分）
- 避免生成过多无意义的短词

## ❌ 问题场景及解决方法

### 场景 1: 看到"窄视口（推荐区域不显示）"

```javascript
[推荐-调试] isNarrowViewport: true
[推荐-调试] isNarrowViewport 变化: 窄视口（推荐区域不显示）
```

**原因**: 浏览器窗口太窄，推荐区域被隐藏了（响应式设计）。

**解决方法**: 
1. 放大浏览器窗口宽度（至少 1200px 以上）
2. 或按 `F11` 全屏查看
3. 或在代码中临时修改：将 `v-if="!isNarrowViewport"` 改为 `v-if="true"`

---

### 场景 2: 看到"从 classificationTags 提取的标签" 或 "从 classificationTag 字符串提取的标签"

```javascript
[推荐-调试] 壁纸 tags: []
[推荐-调试] tags 为空，使用 classificationTags: ["自然", "风景"]
[推荐-调试] 从 classificationTags 提取的标签: ["自然", "风景"]
```

或者：

```javascript
[推荐-调试] 壁纸 tags: []
[推荐-调试] 尝试从 classificationTag 字符串提取: "自然|风景"
[推荐-调试] 从 classificationTag 字符串提取的标签: ["自然", "风景"]
```

或者（最坏情况）：

```javascript
[推荐-调试] 壁纸 tags: []
[推荐-调试] ⚠️ 壁纸没有 tags、classificationTags 或 classificationTag，无法生成推荐
```

**原因**: 
- 壁纸没有 `tags` 字段
- 降级使用 `classificationTags` 数组或 `classificationTag` 字符串

**解决方法**:
1. 在后端为壁纸添加 `tags`、`classificationTags` 或 `classificationTag` 字段
2. 或选择其他有标签的壁纸查看

**检查后端数据**:

确认返回的数据包含以下字段之一：

{
  "data": {
    "items": [
      {
        "id": "1",
        "tags": ["风景", "自然"],  // ← 优先使用（数组）
        "classificationTags": ["风景", "自然"],  // ← 降级使用（数组）
        "classificationTag": "自然|风景",  // ← 降级使用（字符串，支持 | / , ， 、）
        ...
      }
    ]
  }

### 场景 3: 看到"过滤后的有效标签: []" 或 "标签可能是 ID"

```javascript
[推荐-调试] 壁纸 tags 原始数据: ["a754117de8bf", "b863259cf1a2"]
[推荐] 过滤掉疑似ID的标签: "a754117de8bf"
[推荐] 过滤掉疑似ID的标签: "b863259cf1a2"
[推荐-调试] 过滤后的有效标签: []
[推荐-调试] ⚠️ 没有有效的标签，无法生成推荐
[推荐-调试] 原始标签: ["a754117de8bf", "b863259cf1a2"]
[推荐-调试] 提示: 标签可能是 ID 而不是文本，请检查后端数据
```

**原因**: 后端返回的 `tags` 是 **ID 数组**，而不是**文本标签**。

**错误的数据格式**:
```json
{
  "tags": ["a754117de8bf", "b863259cf1a2"]  // ← ID，不是文本
}
```

**正确的数据格式**:
```json
{
  "tags": ["风景", "自然", "山水"]  // ← 文本标签
}
```

**解决方法**:

1. **修改后端代码**（推荐），返回标签的文本而不是 ID：
   ```javascript
   // ❌ 错误
   const tags = wallpaper.tags.map(tag => tag.id)
   
   // ✅ 正确
   const tags = wallpaper.tags.map(tag => tag.name)
   ```

2. **检查后端数据**:
   ```bash
   curl "http://127.0.0.1:3001/api/v1/pc-wallpapers?page=1&pageSize=1"
   ```

3. **查看详细说明**: [TAG_ID_ISSUE.md](TAG_ID_ISSUE.md)

**前端已添加自动过滤**，会过滤掉：
- 超过 20 字符的标签
- 纯数字标签
- 长字母数字组合（如 `a754117de8bf`）
- 包含特殊字符的标签

但根本解决方法是**修改后端返回正确的数据**。

### 场景 4: 看到"请求失败" - 400 Bad Request

```javascript
[推荐-调试] 📡 开始请求: tag="风景" page=1
[推荐-调试] ❌ 请求失败: Error: Network Error
```

**原因**: 后端接口请求失败。

**解决方法**:
1. 检查后端服务是否启动：`node check-backend.cjs`
2. 检查 Network 面板是否有请求（状态码应为 200）
3. 检查后端是否支持 `tag` 参数

---

### 场景 4: 看到"请求成功，返回 0 条数据"

```javascript
[推荐-调试] ✅ 请求成功，返回 0 条数据
[推荐-调试] 当前关键词无结果，切换到下一个
```

**原因**: 后端对该关键词返回了空结果。

**可能的原因**:
1. 数据库中没有匹配该标签的壁纸
2. 后端未实现 `tag` 参数的模糊搜索（精确匹配导致找不到）

**解决方法**:
1. 添加更多测试数据到数据库
2. 确认后端实现了模糊搜索：
   ```javascript
   // ❌ 错误：精确匹配
   { tags: tag }
   
   // ✅ 正确：模糊匹配
   { tags: { $regex: tag, $options: 'i' } }
   ```

---

### 场景 5: 看到"已获得足够推荐（16条），停止加载"

```javascript
[推荐-调试] 累计推荐: 16 条
[推荐-调试] 已获得足够推荐（16条），停止加载
```

**说明**: 这是正常的优化行为，获得 16 条推荐后会自动停止，避免发送过多请求。

---

### 场景 6: 看到"提取的关键词（已限制为前 3 个）"

```javascript
[推荐-调试] 提取的关键词（已限制为前 3 个）: ["动漫美女", "动漫", "美女"]
[推荐-调试] （原始关键词数量: 8 个）
```

**说明**: 为避免同时发送过多请求，关键词数量被限制为前 3 个（按长度排序，优先使用完整标签）。

---

### 场景 7: 看到"去重后有 0 条新数据"

```javascript
[推荐-调试] ✅ 请求成功，返回 16 条数据
[推荐-调试] 去重后有 0 条新数据
```

**原因**: 返回的数据都是重复的（已经存在于推荐列表中）。

**解决方法**: 这是正常的，会自动切换到下一个关键词继续加载。

---

### 场景 8: 没有任何日志输出

**原因**: 
1. 组件根本没有加载
2. 路由配置错误
3. Console 被清空了

**解决方法**:
1. 确认 URL 是否正确（例如 `/pc-wallpapers/1`）
2. 刷新页面并重新查看 Console
3. 检查是否有 JavaScript 错误（红色报错）
